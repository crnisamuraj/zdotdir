# Plan: Restructure to declarative home-mirroring dotfiles repo

## Context
The current repo lives directly at `~/.config/zsh` and only manages zsh config. The `~/.zshenv`
bootstrap is generated by the installer rather than tracked in git. The goal is to restructure
into a general-purpose dotfiles repo where the directory layout mirrors `$HOME` ‚Äî so any file at
`some/path` in the repo installs to `~/some/path`. This eliminates generated files, makes
`.zshenv` a first-class tracked file, and lets the repo grow beyond zsh in the future.

### `.installignore` content
One entry per line, exact filename/path match (no globs). `#` for comments.

> `.git` is omitted ‚Äî `git ls-files` never lists it. New repo meta-files (e.g.
> `CLAUDE.md`, `.claude/`) should be added here if they ever get tracked.

## Steps

### 1. Move zsh config files into `.config/zsh/`
Move all tracked zsh dotfiles from repo root into a `.config/zsh/` subdirectory.
The existing `.zshenv` is the **ZDOTDIR env file** (sets history, XDG vars) ‚Äî not the
bootstrap. It moves into `.config/zsh/`; the bootstrap `.zshenv` is created fresh in step 2.

- `.zprofile` ‚Üí `.config/zsh/.zprofile`
- `.zshenv` ‚Üí `.config/zsh/.zshenv`  *(ZDOTDIR env, not bootstrap)*
- `.zshrc` ‚Üí `.config/zsh/.zshrc`
- `.zsh_plugins.txt` ‚Üí `.config/zsh/.zsh_plugins.txt`
- `.zshrc.d/` ‚Üí `.config/zsh/.zshrc.d/`

Use `git mv` to preserve history.

### 2. Add `.zshenv` at repo root (bootstrap)
Create `.zshenv` at the repo root with the bootstrap content (previously generated):

```zsh
#!/bin/zsh
#
# ~/.zshenv ‚Äî bootstraps ZDOTDIR so Zsh finds the rest of the config.
#

export XDG_CONFIG_HOME=${XDG_CONFIG_HOME:-$HOME/.config}
export XDG_DATA_HOME=${XDG_DATA_HOME:-$HOME/.local/share}
export XDG_CACHE_HOME=${XDG_CACHE_HOME:-$HOME/.cache}
export ZDOTDIR=${ZDOTDIR:-$XDG_CONFIG_HOME/zsh}

[[ -r "$ZDOTDIR/.zshenv" ]] && source "$ZDOTDIR/.zshenv"
```

### 3. Rewrite `install.zsh`

Key changes:
- **Remove** `TARGET_DIR` / custom path argument ‚Äî always installs relative to `$HOME`
- **Remove** `install_zshenv()` ‚Äî `.zshenv` is now installed by the main file loop like any other file
- **Rename** `install_config_files()` ‚Üí `install_files()`, base path is `$HOME`
- Read exclusions from `.installignore` (one entry per line, exact match, `#` comments) instead of a hardcoded list in the script
- macOS deps and atomic staging logic unchanged

Simplified installer flow:
```
install_macos_deps   (unchanged)
install_files        (walk git ls-files ‚Üí stage to tmpdir ‚Üí mv to $HOME/<rel>)
```
